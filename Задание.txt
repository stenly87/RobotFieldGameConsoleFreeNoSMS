Мини-игра
1) через первый ввод в консоль ряда цифр создается поле 5х5, каждая ячейка
поля соответствует определенной цифре (
1 - свободная ячейка, 
2 - стена,
3 - старт робота, 
4 - целевая точка)
4111211122111221112231122
tooox
oooxx
oooxx
oooxx
rooxx
первый ввод цифр обрабатывается без задержки

2) через ввод в консоль последующих цифр, они разбиваются на команды движения робота (1 - вверх, 2 - вниз, 3 - влево,
4 - вправо)
1111 - движение на 4 ячейки вверх
1234 - движение вверх, вниз, влево, вправо

3)получается, ряд цифр разбивается на отдельные цифры, на каждую из которых создается команда, передает в исполнитель
команд, который отрабатывает по 1 команде в секунду

Проектируем:
будет 2 исполнителя, которые принимают команды.
первый исполнитель работает без задержки, рисует поле (FieldCommander)
второй исполнитель рисует с задержкой, рисует движение робота (RobotCommander)

этих исполнителей мы можем стратегией заменять в каком-нибудь основном классе для исполнения команд (MainCommander).

Команды для робота сделаны через интерфейс ICommand c методом Execute

у поля будет свой класс Field с массивом, в котором будут цифры, введенные пользователем

у Робота будет свой класс Robot с текущими координатами

классы робота и поля будет синглтонами (одиночками)

Ввод цифр будет производиться через класс Controller, который просто запрашивает строку, разбивает ее на отдельные символы и передает их в исполнителя (MainCommander)
